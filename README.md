# 氣泡排序法

### 簡述原理
氣泡排序法（Bubble Sort）是透過相鄰元素兩兩比較，若前一個元素比後一個元素大，就交換兩者位置。  
每一輪比較後，較大的元素會逐漸往右移動，就像氣泡往上浮一樣，因此稱為氣泡排序法。  
重複執行多輪後，整個數列就會由小到大完成排序。

### 複雜度分析
- 最佳時間複雜度：O(n)
- 平均時間複雜度：O(n^2)
- 最差時間複雜度：O(n^2)
- 空間複雜度：O(1)
- 穩定性：穩定排序

### 模擬實作內容
在本次模擬程式中，氣泡排序法會從左到右依序比較相鄰長條。  
若左邊元素大於右邊元素，兩者就交換位置，並以動畫方式顯示交換過程。  
使用者可透過開始、暫停、重設與速度調整功能，觀察每一輪比較與交換的結果，並搭配比較次數、交換次數與回合數的統計資訊，更清楚理解演算法運作流程。

# 選擇排序法

### 簡述原理
選擇排序法（Selection Sort）是每一輪從尚未排序的資料中找出最小值，然後與目前未排序區間的第一個元素交換。  
經過第一輪後，最小值會被放到最前面；第二輪再從剩下的元素中找第二小的值，依此類推。  
此方法的特色是每一輪只做一次交換，因此交換次數通常較少。

### 複雜度分析
- 最佳時間複雜度：O(n^2)
- 平均時間複雜度：O(n^2)
- 最差時間複雜度：O(n^2)
- 空間複雜度：O(1)
- 穩定性：不穩定排序

### 模擬實作內容
在模擬畫面中，選擇排序法會先標示目前搜尋範圍，再逐一比較元素找出最小值。  
找到最小值後，系統會將其與目前排序位置進行交換，並更新畫面中的柱狀圖順序。  
透過動畫與統計資料，能明顯看出此方法雖然比較次數多，但交換次數相對較少。

# 插入排序法

### 簡述原理
插入排序法（Insertion Sort）是將資料分成「已排序區」與「未排序區」，每次從未排序區取出一個元素，插入到已排序區中的正確位置。  
這個方法很像整理撲克牌時，將新拿到的牌插入手中原本已排好的牌列中。  
當資料筆數不多，或原本資料已接近排序完成時，插入排序法通常會有不錯的效率。

### 複雜度分析
- 最佳時間複雜度：O(n)
- 平均時間複雜度：O(n^2)
- 最差時間複雜度：O(n^2)
- 空間複雜度：O(1)
- 穩定性：穩定排序

### 模擬實作內容
在本系統中，插入排序法會從第二個元素開始，將目前元素與前方已排序區間進行比較。  
若前方元素較大，就向右移動，直到找到適合插入的位置為止。  
透過動畫可清楚看到元素逐步向前插入的過程，也能配合比較次數與交換/移動次數觀察其特性。

# 合併排序法

### 簡述原理
合併排序法（Merge Sort）採用分治法（Divide and Conquer）的概念。  
先將原始數列不斷對半切分，直到每個子數列只剩下一個元素，再將這些小數列依照大小順序逐步合併回去。  
因為合併時每次都能有效率地比較兩個已排序的子數列，所以整體效率比前面三種基本排序法更好。

### 複雜度分析
- 最佳時間複雜度：O(n log n)
- 平均時間複雜度：O(n log n)
- 最差時間複雜度：O(n log n)
- 空間複雜度：O(n)
- 穩定性：穩定排序

### 模擬實作內容
在模擬程式中，合併排序法會先將資料遞迴分割，再依序進行合併。  
畫面會顯示元素在分割與合併過程中的變化，讓使用者能觀察分治法的運作方式。  
此演算法雖然需要額外空間儲存暫時資料，但在大量資料排序時能展現較佳效率。

# 快速排序法

### 簡述原理
快速排序法（Quick Sort）同樣屬於分治法。  
它會先選定一個基準值（pivot），再將資料重新排列，使得比基準值小的元素放到左邊，比基準值大的元素放到右邊，接著再對左右兩側子數列重複進行相同操作。  
由於每次都能快速縮小問題範圍，因此平均情況下具有很好的排序效率。

### 複雜度分析
- 最佳時間複雜度：O(n log n)
- 平均時間複雜度：O(n log n)
- 最差時間複雜度：O(n^2)
- 空間複雜度：O(log n)
- 穩定性：不穩定排序

### 模擬實作內容
在本次模擬中，快速排序法會先選擇一個 pivot，並將其他元素依大小分配到左右兩側。  
排序過程中，畫面會動態呈現基準值比較、元素交換與區間遞迴處理的情形。  
使用者可以透過速度控制與暫停功能，仔細觀察快速排序法如何利用分區方式提升整體排序效率。

# 學習心得

透過這次的排序演算法報告與實作，我不僅複習了從基礎到進階的五種排序演算法原理，更深刻體會到「視覺化」對於理解抽象邏輯的幫助。

在撰寫程式碼的過程中，我發現將演算法的執行步驟轉換為動畫是一大挑戰。為了讓「比較（藍色）」、「交換（紅色）」以及「已排序（綠色）」的狀態能精準呈現，我必須非常清楚每個排序法在每一輪迴圈中的確切狀態與變數變化。特別是在實作**合併排序**與**快速排序**這兩種基於分治法（Divide and Conquer）的演算法時，如何將遞迴（Recursion）的過程具象化，讓我對遞迴呼叫的堆疊（Stack）運作有了更具體的認識。

此外，藉由加入「比較次數」與「交換次數」的統計功能，我也從實務數據中驗證了課本上的時間複雜度理論。例如，可以明顯觀察到在相同資料量下，氣泡排序的交換次數遠高於選擇排序；而快速排序在處理亂序資料時，整體的執行步數與效率確實大幅優於前面三種 O(n²) 的基礎排序法。

總結來說，這次的報告讓我從單純的「背誦演算法」，進階到了「實作、觀察並分析」的層次，也讓我學到了如何透過寫程式，將生硬的程式邏輯轉化為互動介面，是一次非常有成就感的學習報告。
